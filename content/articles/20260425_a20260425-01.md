# DeepSeek V4が変える、AIツール選びの足元

中国発のAI企業 DeepSeek が、最新フラッグシップモデル「V4」を発表しました。デザイナーが直接触るツールではありません。けれど、私たちが普段使う Claude Code や Cursor、Codex の「裏」が、ここから動き始めます。今日読み終わるころには、なぜそれが他人事ではないかが見えるはずです。

## 何が起きたか

DeepSeek は4/24、「DeepSeek V4 Preview」を公開し、即日オープンソース化しました（[出典：DeepSeek API Docs](https://api-docs.deepseek.com/news/news260424)）。

ラインナップは2つです。

- V4-Pro：1.6兆パラメータ（うち49Bが活性化）
- V4-Flash：284Bパラメータ（13B活性化）

両モデルとも 100万トークンの文脈長に対応しました。ハイブリッド注意機構（Hybrid Attention）と呼ぶ独自設計で、長い会話やコードベースを丸ごと渡しても精度を保つ方向に振っています。

価格も特徴的です。DeepSeek 公式APIで V4 Flash は入力 $0.04 / 100万トークン、出力 $0.10 / 100万トークン。OpenRouter など他経由でも入力 $0.14、出力 $0.28 程度にとどまります（[出典：DeepSeek 公式 Models & Pricing](https://api-docs.deepseek.com/quick_start/pricing)）。フロンティアモデルの料金とは一桁、場合によっては二桁ちがいます。

そして公式は明言しています。「V4 は Claude Code、OpenClaw、OpenCode といった主要 AI エージェントとシームレスに統合される」と。社内評価では V4-Pro-Max が Claude Sonnet 4.5 を上回り、Opus 4.5 に迫ったと主張しています。

## なぜデザイナーに関係あるか

「コーディングモデルの話でしょ」と思われたかもしれません。私もそう思いました。

ただ、ここ1か月の流れを並べると、見え方が変わります。Anthropic は Claude Design を出し（4/17）、OpenAI は Codex でページ要素に直接コメントできる機能を入れ（4/21）、Figma はキャンバスをエージェントに開放しました。デザインツールの「中身」が、コーディングエージェントと地続きになり始めています。

つまり、私たちが Claude Design や Codex に「このカードを20%広げて」と頼んだとき、その指示を解釈しているのは、こうした基盤モデルです。

DeepSeek V4 が示したのは2つの事実です。1つ目は、フロンティアモデルとオープンソースの差が、もう一段詰まったこと。2つ目は、API原価が一桁安い選択肢が出てきたこと。

これは、私たちが触るデザインAIの料金や応答速度に、半年〜1年単位で効いてきます。

## 明日からできる一歩

今日いますぐツールを乗り換える話ではありません。私が選んだのは、次の3つです。

1. 自分が使っているAIツール（Claude Code、Cursor、Codex など）の「裏のモデル」が何かを一度だけ確認する
2. モデル切り替え機能があるツールなら、設定画面のモデル一覧をスクショで残しておく（半年後に比べる用）
3. 「100万トークン文脈」が自分の仕事に効く場面を1つだけ書き出す（例：ブランドガイド全部を一度に読ませて整合チェック）

3つ目が一番大事です。長い文脈を活かす仕事は、これまで「分割して食わせる前提」で組んできたはずです。前提が変わるなら、ワークフローも変わります。

## 私（ciro）の視点

DeepSeek V4 が「Claude Sonnet 4.5 を超えた」かどうかは、ベンチマークの取り方次第で揺れます。鵜呑みにする話ではありません。

それでも私が今回目を留めたのは、価格と文脈長です。デザイナーの仕事は、参照素材の量と整合性が品質を決めます。安く、長く、まとめて読める基盤が増えるなら、私たちが組み立てるAIワークフローの「土台コスト」が下がります。

地味ですが、この地味さに気づける人が、半年後に強いと思っています。

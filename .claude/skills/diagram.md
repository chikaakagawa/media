# diagram

## 目的
note記事の内容を視覚的に補完するフロー図・比較表・構造図のプロンプトを生成する。
writer-article の初稿完成後に呼ぶ。

## 生成するもの
記事の内容に応じて以下から選択:

- **フロー図**: 手順・プロセス・ループを可視化（Mermaid形式）
- **比較表**: Before/After・ツール比較・選択肢の整理（Markdown表）
- **構造図**: 概念・関係性の整理（Mermaid形式 or Canva用レイアウト指示）

## 判定基準
| 記事の種類 | 推奨図 |
|---|---|
| 手順・ステップ系 | フロー図 |
| Before/After・比較系 | 比較表 |
| 仕組み・概念説明系 | 構造図 |
| 体験談・エッセイ系 | 図不要（テキストで十分） |

## 出力フォーマット

### Mermaid（フロー図・構造図）
```mermaid
（コードブロック形式で出力）
```
→ note はMermaid非対応のため、**Canvaで手作りする際のレイアウト指示**も必ず添える。

### 比較表
Markdown表で出力。noteにそのまま貼れる形式。

### Canvaレイアウト指示
```
【Canvaで作る場合】
- 図の種類: 〇〇
- ボックス数: N個
- 配置: 左→右 / 上→下
- 色: ciroトーン（ベージュ・オフホワイト・グレー＋1アクセント）
- テキスト量: 各ボックス1行以内
```

## 制約
- 図は1記事1点まで（情報過多を避ける）
- 専門用語はキャプションで補足
- ciroの世界観（シンプル・余白多め）を維持
